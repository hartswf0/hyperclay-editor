Guiding Principles & Supporting Structures

Studio Reality & Flexibility

  • "There is no single pathway to the defense." Timeline is a guide.
  • Support for deviation: Studio projects, research pivots, mentorship.
  • Embrace iteration: Proposals, Prototypes, IRBs evolve (V1 → V2+).
  • Capture all versions in shared folder with advisor.
  • Value process: Final project reflects the journey (DM ethos).

Program Logic & Dependencies

  • ✅ = Mandatory institutional milestone.
  • IRB Discovery (T2) → Summer Fieldwork (TSummer).
  • IRB Evaluation (T3) → T4 Evaluation & Publication.
  • Design Criteria (T3) → Defense Rubric (T4).
  • Final Project = Professional calling card (Depth over volume).

Cohort Logic & Mentorship

  • (Yr 1) & (Yr 2) coexist.
  • Shared blends cohorts.
  • Fruits mentor Sprouts (e.g., IRB process, examples).

Feedback, Portfolio & Clarity

  • Structured feedback collection at key points.
  • Portfolio support embedded in curriculum.
  • Aim for temporal clarity ("Temporal GPS") for students.

Key Iterative Elements

  • IRB Chain: Discovery → Summer Data → Evaluation → T4 Testing/Publishing.
  • Versioning: Proposals (v1→v2), Prototypes (v1→v2), Website (v1→final).
  • Criteria-Rubric Link: T3 criteria form the basis for T4 defense assessment.
